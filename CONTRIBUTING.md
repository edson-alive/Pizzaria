# Contributing to Pizzaria

Thank you for considering contributing to the Pizzaria project! We appreciate your help and feedback. Please follow the guidelines below to ensure a smooth contribution process.

## Code Style
- Follow consistent coding conventions throughout the project.
- Use [Prettier](https://prettier.io/) for formatting your code.
- Write clean and understandable code. Avoid unnecessary complexity.

## Commit Conventions
- Use clear and descriptive commit messages.
- Follow the format: `type(scope): subject` where `type` can be `feat`, `fix`, `docs`, etc. Examples:
  - `feat(auth): add JWT authentication`
  - `fix(cart): resolve issue with item quantity`
- Reference issues in your commits when applicable, using `#issue_number`.

## Pull Request Process
- Fork the repository and create your feature branch off `main`.
- Ensure your branch is up to date with the latest changes from `main`.
- Open a pull request (PR) and provide a clear description of your changes.
- Tag relevant team members for review.
- Respond to feedback and make necessary changes. Your PR will be merged once it's approved.

## Development Setup
1. **Clone the repository**  
   `git clone https://github.com/edson-alive/Pizzaria.git`

2. **Install dependencies**  
   `cd Pizzaria`  
   `npm install`  

3. **Run the application**  
   `npm start`  

4. **Run tests**  
   `npm test`  

5. **Create a new feature branch**  
   `git checkout -b your-feature-name`

Thank you for your contributions! Happy coding!
